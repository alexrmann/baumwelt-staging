Template Project

This should be a 1-2 sentence introductory statement about the project.

This is the project body. Generally, it should be an approximately 2-3 paragraph description of the project.

File Name

Remove the _ from the front of the template when naming the project. The underscore tells Jekyll not to process and include the file in the site.

Use hyphens (-) to separate words in the file name.

The file extensions should always be .md to denote its use of Markdown syntax. HTML (.html) is also valid, but it will not accept Markdown syntax and must use valid HTML markup.

Front Matter

The project title is the project name. Each word should be capitalized.

The category should only have a single entry. It is top-level hierarchical and general grouping. Each project should exist inside of a single category. The categories should be kept broad to be as inclusive as possible.

The featured-img is a file path. Change file-name.ext to match the file name and extension.

The featured-img-alt is a short, objective description of the image for screen readers.

The client and location tags are self explanatory.

Role implies Baumwelt’s role in the project (e.g., master planning, landscape architecture, construction administration).

Status is for the phase of the project (e.g., in-progress, complete).

Tags accepts multiple single words or quote-enclosed multi-word labels (e.g., “habitat restoration”). You can use and create as many new tags as you need.

Finally, excerpt is a very short introductory description of the project. It displays with each project listed on the Projects page.

General authoring guidelines

The title is a top-level heading (#). No heading in the body should be higher than second-level (##). Avoid using headings deeper than the fourth-level (####).

If you choose not to fill in a front matter field (esp. featured-img and featured-img-alt) it is best to remove any placeholder content and leave the field blank.
