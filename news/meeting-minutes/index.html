Content Templates

I have emailed you standard templates for creating news posts and project pages. We can review them together when we next meet, but I encourage you to use them now.

Each template file describes how to use it. They each have two key properties:

  • The content is written using the Markdown syntax
  • Front Matter defines data about the content

Markdown

All news posts and project pages will need to be written using the Markdown syntax. Jekyll (our site generator) interprets Markdown files (.md) and converts them to HTML.

While Word files can be converted to Markdown, writing files directly in Markdown is likely faster and less error-prone. Markdown is also easy to learn. The Markdown Guide is an excellent resource for learning Markdown.

I recommend using a simple text editor to author Markdown. Notepad will work, but something like Sublime Text provides syntax highlighting, making it easier to both read and write.

  • Start here: Getting Started
  • Most content can be created with the Basic Syntax.
  • More complex formatting options are outlined in the Extended Syntax

Front Matter

These templates will also include “front matter” where data about the content can be defined (e.g., title, author, tags, etc.). Standard data fields have semantic meaning for the site, including auto-populating sections of page layouts, as well as site-wide content organization.
