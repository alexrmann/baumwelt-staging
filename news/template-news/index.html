File Name

Remove the underscore (_) from the front of the template when naming the project. The underscore tells Jekyll not to process and include the file in the site.

You may also author posts in the _drafts folder. When the post is ready to publish, move it to the _posts folder.

Every post must include the year, month, and day (i.e. YYYY-MM-DD) and a lowercase, hyphen-separated name (e.g., post-name). If no title is provided in the front matter, the name of the file (excluding the date) will be used as the title automatically. Use a hyphen (-) to separate each part of the date and title in the file name (i.e. YYYY-MM-DD-post-name).

The file extensions should always be .md to denote its use of Markdown syntax. HTML (.html) is also valid, but it will not accept Markdown syntax and must use valid HTML markup.

Front Matter

The project title is the project name. Each word should be capitalized. The author is a hyphen (-) separated first and lastname.

The featured-img is a file path. Change filename.ext to match the file name and extension.

The featured-img-alt is a short, objective description of the image for screen readers.

Tags accepts multiple single words or hyphen-separated (-) multi-word labels (e.g., habitat-restoration). You can use and create as many new tags as you need. Enclosing brackets ([ ]) are optional, however if you use them, you must separate each entry with a comma (,). Here they are used to visually distinguish the tag array, which is a series of entries.

SEO

Along with the post title, these front matter categories are used by a Jekyll plugin to provide search engines and link sharing with descriptive content about the post.

Image is an optional field. It can be the same file as the featured image, or a different image optimized for link thumbnails.

Finally, excerpt is a very short introductory description of the project. It displays with each project listed on the News page. If left blank, it will use the first paragraph in the post body.

General authoring guidelines

The title is a top-level heading (#). No heading in the body should be higher than second-level (##). Avoid using headings deeper than the fourth-level (####).

If you choose not to fill in a front matter field (esp. featured-img and featured-img-alt) it is best to remove any placeholder content and leave the field blank.
